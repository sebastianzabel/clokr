## Prisma models (from packages/db/prisma/schema.prisma)

```prisma
model SaldoSnapshot {
  id              String          @id @default(uuid())
  employeeId      String
  periodType      SaldoPeriodType // MONTHLY | YEARLY
  periodStart     DateTime        @db.Date
  periodEnd       DateTime        @db.Date
  workedMinutes   Int
  expectedMinutes Int
  balanceMinutes  Int
  carryOver       Int             // cumulative carry-over up to periodEnd (minutes)
  closedAt        DateTime        @db.Timestamptz
  employee        Employee        @relation(fields: [employeeId], references: [id], onDelete: Restrict)

  @@unique([employeeId, periodType, periodStart])
}

model OvertimeAccount {
  id           String   @id @default(uuid())
  employeeId   String   @unique
  balanceHours Decimal  @db.Decimal(7, 2) @default(0)  // live current saldo
  employee     Employee @relation(fields: [employeeId], references: [id], onDelete: Cascade)
}
```

Notes for executor:
- `SaldoSnapshot` has NO `deletedAt` field and NO direct `tenantId`. Tenant scoping MUST go
  through the `employee` relation: `where: { employee: { tenantId } }`.
- `OvertimeAccount` has NO `deletedAt` field. Tenant scoping goes through `employee`.
- `SaldoSnapshot.carryOver` is stored as **minutes** (Int), matching `balanceMinutes`.
- `OvertimeAccount.balanceHours` is a Prisma `Decimal` — convert with `Number(x)` before math.

    ```ts
    // GET /api/v1/dashboard/overtime-trend — Team overtime saldo trend (last 6 months)
    app.get("/overtime-trend", {
      schema: { tags: ["Dashboard"], security: [{ bearerAuth: [] }] },
      preHandler: requireAuth,
      handler: async (req) => {
        const tenantId = req.user.tenantId;

        // 6-month window (inclusive of current month)
        const sixMonthsAgo = new Date();
        sixMonthsAgo.setUTCMonth(sixMonthsAgo.getUTCMonth() - 5);
        sixMonthsAgo.setUTCDate(1);
        sixMonthsAgo.setUTCHours(0, 0, 0, 0);

        // Query 1: SUM(carryOver) grouped by periodStart, MONTHLY only, tenant-scoped.
        // Prisma groupBy does not support relation filters in `where`, so first fetch the
        // employee IDs for this tenant, then group by periodStart over those IDs.
        const employees = await app.prisma.employee.findMany({
          where: { tenantId, user: { isActive: true } },
          select: { id: true },
        });
        const employeeIds = employees.map((e) => e.id);

        const grouped = employeeIds.length === 0
          ? []
          : await app.prisma.saldoSnapshot.groupBy({
              by: ["periodStart"],
              where: {
                employeeId: { in: employeeIds },
                periodType: "MONTHLY",
                periodStart: { gte: sixMonthsAgo },
              },
              _sum: { carryOver: true },
              orderBy: { periodStart: "asc" },
            });

        const snapshots = grouped.map((g) => ({
          month: g.periodStart.toISOString().slice(0, 10), // "YYYY-MM-DD" (always day 01)
          teamCarryOverMinutes: g._sum.carryOver ?? 0,
        }));

        // Query 2: SUM(balanceHours * 60) across all active employees' OvertimeAccounts.
        const accounts = employeeIds.length === 0
          ? []
          : await app.prisma.overtimeAccount.findMany({
              where: { employeeId: { in: employeeIds } },
              select: { balanceHours: true },
            });
        const currentTeamBalanceMinutes = Math.round(
          accounts.reduce((sum, a) => sum + Number(a.balanceHours) * 60, 0),
        );

        return { snapshots, currentTeamBalanceMinutes };
      },
    });
    ```

    Rationale (embed in plan, NOT in a code comment):

    - Do NOT filter on `exitDate: null` for the snapshot query — historical carry-over for
      employees who have since left MUST remain visible for past months. The
      `OvertimeAccount` query uses `isActive` because it reflects current state.
    - Do NOT add `deletedAt: null` to SaldoSnapshot or OvertimeAccount — neither model has
      that column (see schema.prisma lines 321–359).

    if (overtimeChartEl) {
      // Build a lookup of snapshot month → teamCarryOverMinutes
      const snapshotByMonth = new Map<string, number>();
      for (const s of overtimeTrend.snapshots) {
        // API returns "YYYY-MM-DD" (day is always 01); key by "YYYY-MM"

          const snap = snapshotByMonth.get(months[i].month);
