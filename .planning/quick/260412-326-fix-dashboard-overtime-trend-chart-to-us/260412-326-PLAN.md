<objective>
Fix the "Überstunden-Trend" line chart on the dashboard so the last data point matches the stored saldo shown in the summary bar. The chart currently builds a naive running cumulative from team-aggregated `/reports/monthly` data starting at 0, which ignores pre-period carry-over and aggregates over all employees instead of the current user. It therefore diverges from `stats.overtime.balanceHours` (the current user's OvertimeAccount balance).

Purpose: Give the user a trustworthy trend that is anchored in persisted SaldoSnapshots (the same source used by Monatsabschluss), so the trend line naturally reaches the stored saldo at the right edge.

Output:
- Dashboard overtime chart driven by `GET /api/v1/overtime/snapshots/:employeeId` for the current user, using each snapshot's `carryOver` as the cumulative saldo at period end.
- Current (open) month represented by a trailing point that equals `stats.overtime.balanceHours` so the right edge of the chart matches the summary bar.
- No API changes. No schema changes.
</objective>

## API: GET /api/v1/overtime/snapshots/:employeeId
Defined in `apps/api/src/routes/overtime.ts` (around line 883).
Auth: `requireAuth` (JWT).
Returns: `SaldoSnapshot[]` (ordered `periodStart: "desc"`, so the NEWEST snapshot is first).

Shape of each snapshot (Prisma `SaldoSnapshot` model, `packages/db/prisma/schema.prisma` line 321):
```ts
interface SaldoSnapshot {
  id: string;
  employeeId: string;
  periodType: "MONTHLY" | "YEARLY";
  periodStart: string;   // ISO date (YYYY-MM-DD) for @db.Date columns serialized over JSON
  periodEnd: string;     // ISO date (YYYY-MM-DD)
  workedMinutes: number; // Ist within the period
  expectedMinutes: number; // Soll within the period
  balanceMinutes: number;  // worked - expected within the period
  carryOver: number;       // CUMULATIVE saldo AT periodEnd (includes all previous months)
  closedAt: string;        // ISO timestamp
  closedBy: string | null;
  note: string | null;
}
```

Key field for the chart: `carryOver` is the cumulative saldo up to and including `periodEnd`. That is exactly what the trend chart needs per month — no client-side cumulation required.

Filter the response client-side for `periodType === "MONTHLY"` only; drop any YEARLY snapshots to avoid double-counting.

## API: GET /api/v1/dashboard (already used)
Returns `DashboardStats` which contains `overtime.balanceHours: number` — the CURRENT saldo for the logged-in employee from `OvertimeAccount.balanceHours`. This is the authoritative value the chart must converge to at the right edge.

## Auth store
`$authStore.user.employeeId: string | null` — available on the dashboard page, already imported via `import { authStore } from "$stores/auth"`. Guard for null (API-only users have no employeeId — in that case, skip the snapshot fetch and fall back to an empty dataset or a single-point dataset using `stats.overtime.balanceHours`).

   ```ts
   // Load current user's saldo snapshots for the overtime trend
   interface SaldoSnapshotApi {
     periodType: "MONTHLY" | "YEARLY";
     periodStart: string; // YYYY-MM-DD
     periodEnd: string;   // YYYY-MM-DD
     balanceMinutes: number;
     carryOver: number;   // cumulative saldo at periodEnd, in minutes
   }
   let monthlySaldoHours: (number | null)[] = new Array(months.length).fill(null);
   const currentEmployeeId = $authStore.user?.employeeId ?? null;
   if (currentEmployeeId) {
     try {
       const snapshots = await api.get<SaldoSnapshotApi[]>(
         `/overtime/snapshots/${currentEmployeeId}`,
       );
       // Build a map keyed by "YYYY-MM" for fast lookup
       const snapByMonth = new Map<string, SaldoSnapshotApi>();
       for (const s of snapshots) {
         if (s.periodType !== "MONTHLY") continue; // ignore YEARLY rollups
         const key = s.periodStart.slice(0, 7); // "YYYY-MM"
         snapByMonth.set(key, s);
       }
       monthlySaldoHours = months.map((m) => {
         const snap = snapByMonth.get(m.month);
         return snap ? +(snap.carryOver / 60).toFixed(1) : null;
       });
     } catch (err) {
       console.error("Failed to load saldo snapshots for overtime chart:", err);
     }
   }
   ```

   ```ts
   // Overtime trend line chart — driven by SaldoSnapshots (see quick 260412-326)
   if (overtimeChartEl) {
     // For each of the last 6 months, use the snapshot's cumulative carryOver (in hours).
     // For months without a snapshot (typically the current, still-open month),
     // fall back to the authoritative stats.overtime.balanceHours so the last
     // visible point matches the summary bar exactly.
     const currentBalanceHours = stats?.overtime?.balanceHours ?? 0;

     // Fill forward: if a middle month has no snapshot, reuse the previous known value
     // so the line doesn't drop to 0. If NO snapshots exist at all, every point falls
     // back to currentBalanceHours (flat line at the real saldo).
     const filled: number[] = [];
     let lastKnown: number | null = null;
     for (let i = 0; i < monthlySaldoHours.length; i++) {
       const v = monthlySaldoHours[i];
       if (v !== null) {
         lastKnown = v;
         filled.push(v);
       } else {
         // No snapshot for this month — decide fallback:
         // - If this is the LAST bucket (current open month), use currentBalanceHours.
         // - Otherwise, reuse the last known snapshot value (carry forward).
         if (i === monthlySaldoHours.length - 1) {
           filled.push(+currentBalanceHours.toFixed(1));
         } else {
           filled.push(lastKnown ?? +currentBalanceHours.toFixed(1));
         }
       }
     }

     // Guarantee the right-edge point matches the summary bar regardless of whether
     // the current month had a snapshot (defensive: if auto-close ran today, the
     // snapshot's carryOver should already equal the account balance, so this is a
     // no-op in the common case).
     filled[filled.length - 1] = +currentBalanceHours.toFixed(1);

     overtimeChart?.destroy();
     overtimeChart = new Chart(overtimeChartEl, {
       type: "line",
       data: {
         labels,
         datasets: [
           {
             label: "Überstunden kumuliert (h)",
             data: filled,
             borderColor: brandColor,
             backgroundColor: brandColor + "20",
             fill: true,
             tension: 0.3,
             pointRadius: 4,
             pointBackgroundColor: brandColor,
           },
         ],
       },
       options: {
         responsive: true,
         maintainAspectRatio: false,
         plugins: { legend: { display: false } },
         scales: {
           y: { grid: { color: "#f3f4f6" }, ticks: { font: { size: 10 } } },
           x: { grid: { display: false }, ticks: { font: { size: 10 } } },
         },
       },
     });
   }
   ```
