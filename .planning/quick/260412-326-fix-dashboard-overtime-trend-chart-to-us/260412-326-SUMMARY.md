# Quick 260412-326: Fix Dashboard Overtime Trend Chart

**One-liner:** Rewired the Überstunden-Trend chart from naive team-cumulation to per-user SaldoSnapshot.carryOver, with stats.overtime.balanceHours anchoring the right-edge point.

## What Changed

### Before (buggy behavior)

The "Überstunden-Trend" line chart in `loadCharts()` was built from `/reports/monthly` data:
- `reports` aggregated ALL employee rows (team totals, not the current user)
- A running `cumulative` sum started at `0` ignoring any pre-period carry-over
- Result: chart diverged from the `stats.overtime.balanceHours` summary bar value shown to the user

### After (correct behavior)

1. `loadCharts()` now fetches `GET /api/v1/overtime/snapshots/:employeeId` for the currently logged-in user (using `$authStore.user?.employeeId`)
2. MONTHLY snapshots are mapped by `"YYYY-MM"` key into a lookup Map
3. Each of the 6 chart buckets uses the snapshot's `carryOver` field (cumulative saldo at period end, in minutes → converted to hours)
4. The last bucket is always overwritten with `stats.overtime.balanceHours` so the chart's right edge matches the summary bar exactly
5. Months with no snapshot (typically the current open month) fill forward from the last known value; if no snapshots exist at all, all points show `currentBalanceHours` (flat line)

## Edge Cases Handled

| Scenario | Behavior |
|---|---|
| Current open month (no snapshot) | Last point = `stats.overtime.balanceHours` |
| Multiple past months without snapshots | Fill-forward from last known snapshot value |
| No snapshots at all (new employee) | All 6 points = `stats.overtime.balanceHours` (flat line) |
| API-only user (no `employeeId`) | Skip snapshot fetch; all points = `currentBalanceHours` |
| Snapshot fetch fails | `console.error` logged; chart falls back to all-null → fill-forward from `currentBalanceHours` |
