<objective>
Create the FritzBox adapter: a standalone, separately deployable Node 24 TypeScript service
that polls the FritzBox TR-064 API every 60 s (configurable), diffs host presence against
in-memory state, debounces disconnects (6 consecutive absent polls), handles outage gaps
(>=30 min absence triggers synthetic disconnect then reconnect), and POSTs normalized events
to the clokr-api webhook. Packaged as a Docker container added to docker-compose.yml under
an opt-in profile so it never starts without explicit configuration.

Purpose: Phase 25 delivers the FritzBox adapter as the first concrete device adapter for
the WiFi-presence clock-in/out system. All consumer logic lives in clokr-api (plans 25-03
and 25-04); this adapter is a pure event emitter requiring no Prisma, no Fastify, and no
clokr business logic — just TR-064 polling, state diffing, and HTTP webhook posting.

Output:
- apps/fritzbox-adapter/ — full package with TR-064 client, state machine, entry point, tests
- docker-compose.yml — fritzbox-adapter service under profiles: [fritzbox]

FILE: src/tr064.ts — TR-064 SOAP client with manual Digest auth

Export interface FritzDevice: { mac: string, hostname: string, active: boolean, ip: string }.

Export function normalizeMac(raw: string): string — strips all non-hex characters, splits
into 6 two-char bytes, joins with colons, lowercases. E.g. "AA-BB-CC-DD-EE-FF" becomes "aa:bb:cc:dd:ee:ff".

Export class Tr064Client with constructor(url: string, user: string, pass: string).

Implement private async digestFetch(path: string, body: string, soapAction: string): Promise<string>:
  1. First attempt: POST to url+path with SOAPAction header and Content-Type text/xml; charset=utf-8.
     Body is SOAP envelope. If response is 200, return text body directly.
  2. If response is 401 with WWW-Authenticate: Digest header:
     Parse realm, nonce, qop from header using regex.
     Compute MD5 using Node's built-in crypto.createHash("md5"):
       HA1 = md5(user:realm:pass)
       HA2 = md5(POST:path)
       nc = "00000001", cnonce = random 8-char hex from crypto.randomBytes(4)
       response = md5(HA1:nonce:nc:cnonce:qop:HA2) when qop is present
       response = md5(HA1:nonce:HA2) when qop is absent
     Retry with Authorization: Digest header including username, realm, nonce, uri, nc, cnonce, response, qop.
  3. If still non-200 after retry, throw Error with status code and snippet of body.

Implement async getHostList(): Promise<FritzDevice[]>:
  SOAP envelope targeting urn:dslforum-org:service:Hosts:1 action GetHostListPath.
  Path: /upnp/control/Hosts
  SOAPAction: "urn:dslforum-org:service:Hosts:1#GetHostListPath"
  Parse XML response to extract the host list path (a URL string to a secondary XML file).
  Fetch that secondary URL (simple GET, same Digest auth), parse each host entry:
    Extract MAC, HostName, Active ("1" = true), IPAddress.
  Normalize each MAC via normalizeMac before returning.

XML parsing: use regex/string operations on the known FritzBox response structure only.
If GetHostListPath XML parsing is brittle in practice, fall back to a loop over
GetGenericHostEntry(0..N-1) where N is from GetHostNumberOfEntries.

FILE: src/state.ts — In-memory MAC state machine

Constants (exported for testability):
  ABSENT_THRESHOLD = 6              // consecutive absent polls before emitting disconnected
  GAP_THRESHOLD_MS = 30 * 60 * 1000   // 30 min in ms

Interface MacState (not exported — internal):
  { lastState: "connected" | "disconnected", lastSeen: Date, absentCount: number }

Export interface PresenceEvent:
  { macAddress: string, state: "connected" | "disconnected", timestamp: Date }

Export class PresenceStateManager:
  private map: Map<string, MacState> = new Map()

  processHostList(hosts: FritzDevice[], now: Date): PresenceEvent[]
    Build activeMacs = new Set of MAC from hosts where host.active === true.
    events: PresenceEvent[] = []

    For each mac in activeMacs:
      If mac not in this.map (first-ever seen):
        this.map.set(mac, { lastState: "connected", lastSeen: now, absentCount: 0 })
        events.push({ macAddress: mac, state: "connected", timestamp: now })
        continue
      existing = this.map.get(mac)!
      If existing.lastState === "disconnected":
        gapMs = now.getTime() - existing.lastSeen.getTime()
        If gapMs >= GAP_THRESHOLD_MS:
          (synthetic disconnect at last-seen time, then fresh connect)
          events.push({ macAddress: mac, state: "disconnected", timestamp: existing.lastSeen })
          events.push({ macAddress: mac, state: "connected", timestamp: now })
        Else:
          (short gap — reconnect silently, no event emitted to avoid spurious clock-in retries)
      this.map.set(mac, { lastState: "connected", lastSeen: now, absentCount: 0 })

    For each [mac, existing] in this.map where mac is NOT in activeMacs:
      If existing.lastState === "disconnected": continue (already disconnected)
      newAbsentCount = existing.absentCount + 1
      If newAbsentCount >= ABSENT_THRESHOLD:
        events.push({ macAddress: mac, state: "disconnected", timestamp: now })
        this.map.set(mac, { lastState: "disconnected", lastSeen: existing.lastSeen, absentCount: newAbsentCount })
      Else:
        this.map.set(mac, { ...existing, absentCount: newAbsentCount })

    Return events.

  getAll(): Map<string, MacState> — returns this.map for use by /devices snapshot.

Poll function pollOnce() — async, non-throwing:
  Wrapped in try/catch. On success: getHostList, update lastHosts, processHostList, then
  await postEvent for each returned event. On error: log and continue (next poll retries).

Webhook poster postEvent(event: PresenceEvent) — async:
  POST {CLOKR_API_URL}/api/v1/presence/event with JSON body
  { macAddress, state, timestamp: event.timestamp.toISOString() }.
  Headers: Content-Type application/json, Authorization: Bearer {CLOKR_PRESENCE_KEY}.
  On !res.ok: log status and body text. On ok: log mac + state.

Start-up:
  Log "[fritzbox-adapter] Starting. Poll interval: Xs, devices on BIND:PORT"
  Call pollOnce() immediately (do not wait for first interval tick).
  pollTimer = setInterval(() => pollOnce(), pollIntervalMs)

GET /devices HTTP server (always started — admin UI proxy calls it server-side):
  Node built-in http.createServer. GET /devices responds 200 JSON with lastHosts.
  All other paths respond 404. server.listen(devicesPort, devicesBind).
  Log "[fritzbox-adapter] Devices endpoint listening on BIND:PORT".

Graceful shutdown on SIGTERM / SIGINT:
  clearInterval(pollTimer)
  server.close()
  process.exit(0)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| FritzBox to adapter | TR-064 response XML; attacker on LAN could serve a malicious FritzBox response |
| adapter to clokr-api | HTTP webhook POST; Bearer token in Authorization header; must not be leaked in logs |
| LAN to adapter /devices | GET /devices; must be bound to loopback only in production |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-09-01 | Spoofing | TR-064 SOAP response | accept | Adapter runs on the same trusted LAN as the FritzBox; SSL pinning would require cert management not supported by FritzBox. Document that the adapter must run on a trusted network segment. |
| T-25-09-02 | Information Disclosure | CLOKR_PRESENCE_KEY in env | mitigate | Never log the key. index.ts must not print env dump. Docker compose .env file must not be committed — add apps/fritzbox-adapter/.env to .gitignore. |
| T-25-09-03 | Information Disclosure | GET /devices endpoint | mitigate | DEVICES_BIND defaults to "127.0.0.1". Docker port binding uses "127.0.0.1:8765:8765". Admin UI proxy in clokr-api calls it server-side — never exposed to browser directly. |
| T-25-09-04 | Denial of Service | Poll loop uncaught error | mitigate | try/catch in pollOnce() logs and continues; never crashes the process on a single failed poll. setInterval is not cancelled on error. |
| T-25-09-05 | Tampering | XML injection via FritzBox hostname field | accept | Hostname is only logged and passed to admin UI as a display string; never executed or used in DB queries. No eval or template injection risk. |
| T-25-09-06 | Elevation of Privilege | FRITZBOX_PASS stored in compose env | accept | Dev-only docker-compose pattern; production deployments should use Docker secrets or a secrets manager. Document in Dockerfile comments. |
</threat_model>
