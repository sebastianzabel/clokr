Route structure (all require `preHandler: requireRole("ADMIN")`):

**GET /** — list:
```typescript
const sources = await app.prisma.presenceSource.findMany({
  where: { tenantId: req.user.tenantId, deletedAt: null },
  orderBy: { createdAt: "desc" },
  select: { id, name, keyPrefix, adapterUrl, isActive, lastUsedAt, createdAt, updatedAt },
});
return { sources };
```

**POST /** — create:
- Body schema: `z.object({ name: z.string().min(1).max(100), kind: z.literal("FRITZBOX").default("FRITZBOX"), adapterUrl: z.string().url().optional() })`
- Generate: `rawKey = \`clk_\${randomBytes(32).toString("hex")}\``
- Hash: `keyHash = createHash("sha256").update(rawKey).digest("hex")`
- Prefix: `rawKey.substring(0, 12) + "..."`
- Create via `app.prisma.presenceSource.create({ data: { tenantId, name, keyHash, keyPrefix, adapterUrl } })`
- AuditLog: `action: "CREATE", entity: "PresenceSource", entityId: source.id, newValue: { name, keyPrefix, adapterUrl }`
- Return: `{ id, name, keyPrefix, adapterUrl, isActive, createdAt, rawKey }` — rawKey included once

**PATCH /:id** — update:
- Params: `z.object({ id: z.string().uuid() })`
- Body: `z.object({ name: z.string().min(1).max(100).optional(), adapterUrl: z.string().url().nullable().optional(), isActive: z.boolean().optional() }).refine(b => Object.keys(b).length > 0, { message: "Mindestens ein Feld erforderlich" })`
- Find: `app.prisma.presenceSource.findFirst({ where: { id, tenantId: req.user.tenantId, deletedAt: null } })` — 404 if not found
- Capture `oldValue` before update
- Update and write AuditLog: `action: "UPDATE", oldValue: { name: old.name, adapterUrl: old.adapterUrl, isActive: old.isActive }, newValue: updated fields`
- Return updated record (without keyHash, without rawKey)

**DELETE /:id** — soft delete:
- Params: `z.object({ id: z.string().uuid() })`
- Find with `deletedAt: null` — 404 if already deleted or not in tenant
- Update: `{ deletedAt: new Date(), isActive: false }`
- AuditLog: `action: "DELETE", entity: "PresenceSource", entityId: id, oldValue: { name: source.name }`
- Return: `{ success: true }`

**GET /:id/devices** — adapter proxy:
- Find source (deletedAt: null, tenantId scoped) — 404 if not found
- If `source.adapterUrl` is null: `return reply.code(502).send({ error: "Kein Adapter-URL konfiguriert" })`
- `fetch(\`\${source.adapterUrl}/devices\`, { signal: AbortSignal.timeout(5000) })`
- On network error or non-2xx: `return reply.code(502).send({ error: "Adapter nicht erreichbar" })`
- On success: proxy the JSON body directly — `return reply.send(await adapterRes.json())`
- No AuditLog for read-only proxy

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → API | Admin JWT in Authorization header; all routes require ADMIN role |
| API → FritzBox adapter | Outbound fetch to adapterUrl/devices — operator-configured URL, not user-supplied per-request |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-04-01 | Spoofing | POST / key generation | mitigate | `randomBytes(32)` = 256 bits entropy; SHA256 hash stored; raw key returned once and never logged |
| T-25-04-02 | Information Disclosure | GET / list endpoint | mitigate | `keyHash` excluded from SELECT; `rawKey` only in POST response; ADMIN role required |
| T-25-04-03 | Elevation of Privilege | All routes | mitigate | `requireRole("ADMIN")` preHandler on every route; 403 for EMPLOYEE/MANAGER |
| T-25-04-04 | Spoofing (SSRF) | GET /:id/devices proxy | mitigate | `adapterUrl` is operator-set (not user-supplied per-request); ADMIN required to set adapterUrl via PATCH; `AbortSignal.timeout(5000)` prevents hanging; only JSON proxied (no headers leaked) |
| T-25-04-05 | Tampering | PATCH /:id | mitigate | Tenant-scoped lookup before update prevents cross-tenant modification; AuditLog captures oldValue + newValue |
| T-25-04-06 | Repudiation | CREATE/UPDATE/DELETE | mitigate | AuditLog entry with userId, entityId, ipAddress, old+new values on every mutation |
| T-25-04-07 | Information Disclosure | rawKey in response | accept | Raw key returned once in POST response body (HTTPS in production); not stored, not logged; acceptable for API key provisioning flows |
</threat_model>
