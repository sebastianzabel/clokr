<objective>
Implement the presence webhook handler that is the core of Phase 25. A single `POST /api/v1/presence/events` endpoint receives normalized connect/disconnect events from the FritzBox adapter (and future adapters), authenticates via a PresenceSource Bearer key, runs the full shift-window gate and cross-source dedup logic, and writes a TimeEntry or an AuditLog depending on the outcome.

Purpose: Without this handler the WiFi-presence feature produces no clock-in/out records. This is the hub that the schema (25-01), the utilities (25-02), the admin CRUD (25-04), and the FritzBox adapter (25-09) all connect to.

```typescript
import { FastifyInstance } from "fastify";
import { z } from "zod";
import { createHash } from "crypto";
import { normalizeMac } from "../utils/normalize-mac";
import { getCurrentShift } from "../utils/get-current-shift";
import { getTenantTimezone, todayInTz, dateStrInTz } from "../utils/timezone";
import { fromZonedTime } from "date-fns-tz";

// ── Zod schema ────────────────────────────────────────────
const presenceEventSchema = z.object({
  mac: z.string().min(1),
  eventType: z.enum(["connected", "disconnected"]),
  timestamp: z.string().datetime(), // ISO-8601
  adapter: z.string().default("fritzbox"),
});

// SYSTEM user constant used for automated audit actions (no JWT context)
const SYSTEM_USER_ID = "00000000-0000-0000-0000-000000000000";

export async function presenceRoutes(app: FastifyInstance) {
  // POST /events — receive normalized presence event from an adapter
  const isTest = process.env.NODE_ENV === "test";
  app.post("/events", {
    schema: { tags: ["WiFi-Presence"] },
    config: { rateLimit: { max: isTest ? 5000 : 300, timeWindow: "1 minute" } },
    handler: async (req, reply) => {
      // ── 1. Auth: extract Bearer key → SHA256 → PresenceSource lookup ──
      const authHeader = req.headers.authorization;
      if (!authHeader?.startsWith("Bearer ")) {
        return reply.code(401).send({ error: "Presence-Key erforderlich" });
      }
      const rawKey = authHeader.slice(7);
      const keyHash = createHash("sha256").update(rawKey).digest("hex");

      const source = await app.prisma.presenceSource.findUnique({
        where: { keyHash },
      });
      if (!source || source.revokedAt) {
        return reply.code(401).send({ error: "Ungültiger oder widerrufener Presence-Key" });
      }

      const tenantId = source.tenantId;

      // Fire-and-forget: update lastUsedAt on PresenceSource
      app.prisma.presenceSource
        .update({ where: { id: source.id }, data: { lastUsedAt: new Date() } })
        .catch((err) => app.log.error({ err }, "Failed to update PresenceSource.lastUsedAt"));

      // ── 2. Parse + normalize body ──────────────────────
      const body = presenceEventSchema.parse(req.body);
      const mac = normalizeMac(body.mac);
      const eventTime = new Date(body.timestamp);

      // ── 3. MAC → Employee lookup (opt-in + tenant-scoped) ──
      const employee = await app.prisma.employee.findFirst({
        where: {
          tenantId,
          wifiMacs: { has: mac },
          deletedAt: null,
        },
      });

      if (!employee) {
        // Unknown MAC — purgeable presence-only event
        await app.prisma.auditLog.create({
          data: {
            userId: null,
            action: "WIFI_UNKNOWN_MAC",
            entity: "PresenceEvent",
            entityId: null,
            newValue: { mac, eventType: body.eventType, adapter: body.adapter },
            tenantId,
            purgeable: true,
          },
        });
        return reply.code(200).send({ ok: true });
      }

      if (!employee.wifiPresenceEnabled) {
        // Opt-out — purgeable
        await app.prisma.auditLog.create({
          data: {
            userId: null,
            action: "WIFI_OPT_OUT",
            entity: "PresenceEvent",
            entityId: employee.id,
            newValue: { mac, eventType: body.eventType },
            tenantId,
            purgeable: true,
          },
        });
        return reply.code(200).send({ ok: true });
      }

      // ── 4. Tenant config: shift window ─────────────────
      const tz = await getTenantTimezone(app.prisma, tenantId);
      const tenantCfg = await app.prisma.tenantConfig.findUnique({
        where: { tenantId },
        select: { wifiPresenceWindowMinutes: true },
      });
      const windowMinutes = tenantCfg?.wifiPresenceWindowMinutes ?? 15;

      // ── 5. Shift-window gate ────────────────────────────
      const shift = await getCurrentShift(app.prisma, employee.id, eventTime, tz);

      if (!shift) {
        await app.prisma.auditLog.create({
          data: {
            userId: null,
            action: "WIFI_NO_SHIFT",
            entity: "PresenceEvent",
            entityId: employee.id,
            newValue: { mac, eventType: body.eventType, timestamp: body.timestamp },
            tenantId,
            purgeable: true,
          },
        });
        return reply.code(200).send({ ok: true });
      }

      // Convert shift HH:MM strings to absolute UTC timestamps for comparison
      const dateStr = dateStrInTz(eventTime, tz);
      const shiftStart = fromZonedTime(new Date(`${dateStr}T${shift.startTime}:00`), tz);
      const windowMs = windowMinutes * 60_000;

      // Event must be within window of shiftStart OR shiftEnd
      const nearShiftStart =
        eventTime >= new Date(shiftStart.getTime() - windowMs) &&
        eventTime <= new Date(shiftStart.getTime() + windowMs);

      let nearShiftEnd = false;
      if (shift.endTime) {
        const shiftEnd = fromZonedTime(new Date(`${dateStr}T${shift.endTime}:00`), tz);
        nearShiftEnd =
          eventTime >= new Date(shiftEnd.getTime() - windowMs) &&
          eventTime <= new Date(shiftEnd.getTime() + windowMs);
      }

      if (!nearShiftStart && !nearShiftEnd) {
        await app.prisma.auditLog.create({
          data: {
            userId: null,
            action: "WIFI_OUTSIDE_WINDOW",
            entity: "PresenceEvent",
            entityId: employee.id,
            newValue: {
              mac,
              eventType: body.eventType,
              timestamp: body.timestamp,
              shiftStart: shift.startTime,
              shiftEnd: shift.endTime,
              windowMinutes,
            },
            tenantId,
            purgeable: true,
          },
        });
        return reply.code(200).send({ ok: true });
      }

      // ── 6. Resolve date key for TimeEntry ──────────────
      const today = todayInTz(tz); // midnight UTC representing the date in tenant tz

      // ── 7. Cross-source dedup check ────────────────────
      // If a non-WIFI entry already exists for this date, confirm presence only
      const existingNonWifiEntry = await app.prisma.timeEntry.findFirst({
        where: {
          employeeId: employee.id,
          date: today,
          deletedAt: null,
          source: { in: ["NFC", "MANUAL", "CORRECTION"] },
        },
      });

      if (existingNonWifiEntry) {
        await app.prisma.auditLog.create({
          data: {
            userId: null,
            action: "WIFI_PRESENCE_CONFIRMED",
            entity: "TimeEntry",
            entityId: existingNonWifiEntry.id,
            newValue: {
              mac,
              eventType: body.eventType,
              timestamp: body.timestamp,
              note: `MAC ${mac} beobachtet um ${eventTime.toISOString()}`,
            },
            tenantId,
            purgeable: false,
          },
        });
        return reply.code(200).send({ ok: true });
      }

      // ── 8. Clock-in / clock-out transaction ────────────
      if (body.eventType === "connected") {
        // Check for existing WIFI entry (any state) — never create a second entry
        const existingWifiEntry = await app.prisma.timeEntry.findFirst({
          where: { employeeId: employee.id, date: today, deletedAt: null, source: "WIFI" },
        });

        if (existingWifiEntry) {
          // Already clocked in via WIFI — confirm presence, no duplicate
          await app.prisma.auditLog.create({
            data: {
              userId: null,
              action: "WIFI_PRESENCE_CONFIRMED",
              entity: "TimeEntry",
              entityId: existingWifiEntry.id,
              newValue: { mac, timestamp: body.timestamp },
              tenantId,
              purgeable: false,
            },
          });
          return reply.code(200).send({ ok: true });
        }

        // Create new WIFI clock-in inside a transaction (race-condition guard)
        const newEntry = await app.prisma.$transaction(async (tx) => {
          // Re-check inside transaction to guard against concurrent requests
          const concurrent = await tx.timeEntry.findFirst({
            where: { employeeId: employee.id, date: today, deletedAt: null },
          });
          if (concurrent) return null; // lost race — let the other request win

          return tx.timeEntry.create({
            data: {
              employeeId: employee.id,
              date: today,
              startTime: eventTime,
              source: "WIFI",
            },
          });
        });

        if (newEntry) {
          await app.prisma.auditLog.create({
            data: {
              userId: null,
              action: "WIFI_CLOCK_IN",
              entity: "TimeEntry",
              entityId: newEntry.id,
              newValue: { mac, startTime: eventTime.toISOString(), source: "WIFI" },
              tenantId,
              purgeable: false,
            },
          });
        }

        return reply.code(200).send({ ok: true });
      }

      // body.eventType === "disconnected"
      const openWifiEntry = await app.prisma.timeEntry.findFirst({
        where: {
          employeeId: employee.id,
          date: today,
          deletedAt: null,
          source: "WIFI",
          endTime: null,
          isInvalid: false,
        },
        include: { employee: { select: { tenant: { select: { id: true } } } } },
      });

      if (!openWifiEntry) {
        // No open WIFI entry to close — idempotent
        await app.prisma.auditLog.create({
          data: {
            userId: null,
            action: "WIFI_NO_OPEN_ENTRY",
            entity: "PresenceEvent",
            entityId: employee.id,
            newValue: { mac, timestamp: body.timestamp },
            tenantId,
            purgeable: true,
          },
        });
        return reply.code(200).send({ ok: true });
      }

      // Check isLocked (CLAUDE.md audit-proof rule: locked months must not be modified)
      const entryMonth = openWifiEntry.date;
      const closedMonth = await app.prisma.closedMonth.findFirst({
        where: {
          tenantId,
          year: entryMonth.getUTCFullYear(),
          month: entryMonth.getUTCMonth() + 1,
          isLocked: true,
        },
      });
      if (closedMonth) {
        app.log.warn({ entryId: openWifiEntry.id }, "WIFI clock-out blocked: month is locked");
        return reply.code(200).send({ ok: true }); // silent — no user-facing error for adapter
      }

      await app.prisma.$transaction(async (tx) => {
        await tx.timeEntry.update({
          where: { id: openWifiEntry.id },
          data: { endTime: eventTime },
        });

        await app.prisma.auditLog.create({
          data: {
            userId: null,
            action: "WIFI_CLOCK_OUT",
            entity: "TimeEntry",
            entityId: openWifiEntry.id,
            oldValue: { endTime: null },
            newValue: { mac, endTime: eventTime.toISOString() },
            tenantId,
            purgeable: false,
          },
        });
      });

      return reply.code(200).send({ ok: true });
    },
  });
}
```

| API → Database | Prisma writes are inside the trusted API process. No direct DB access from adapters. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-03-01 | Spoofing | POST /events Bearer key | mitigate | SHA256 hash stored at rest; compare `createHash("sha256").update(rawKey)` — constant-time comparison not required here since timing attacks on a 64-hex hash require infeasible pre-image work |
| T-25-03-02 | Tampering | MAC address in request body | mitigate | normalizeMac() canonicalizes format; MAC is looked up against Employee.wifiMacs in DB — attacker can only claim a MAC that is already enrolled to an employee, and they need a valid PresenceSource key |
| T-25-03-03 | Repudiation | Automated clock-in/out without user JWT | mitigate | Every branch writes an AuditLog with `action`, `entityId`, `mac`, and `timestamp`; `userId=null` explicitly marks automated origin; `purgeable=false` on stamp events ensures 10-year retention per §147 AO |
| T-25-03-04 | Information Disclosure | Unknown MAC logged | mitigate | Unknown MAC audit log entry contains MAC address — purgeable after 90 days (DSGVO Art. 5(1)(e) minimization); MAC alone is not strongly identifying without the employee mapping |
| T-25-03-05 | Denial of Service | Adapter flooding /events endpoint | mitigate | Rate limit: 300 req/min per IP (config: rateLimit in presenceRoutes); FritzBox adapter polls 30 s = ~2 req/min per device, far below limit |
| T-25-03-06 | Elevation of Privilege | Presence key used to access other API endpoints | accept | PresenceSource key is only valid for POST /events handler; other routes require JWT (requireAuth middleware) or their own Terminal key — no cross-route privilege |
| T-25-03-07 | Tampering | Race condition on concurrent clock-in | mitigate | Clock-in wrapped in `$transaction` with re-check inside transaction before create; concurrent second request sees existing entry and returns without creating a duplicate |
