<objective>
Fix the Docker seed compilation for pnpm@10 + Prisma 7 compatibility (GitHub #119).

Purpose: The current `Dockerfile` silently suppresses TypeScript compile errors (`2>/dev/null || true`), meaning `dist/seed.js` may not exist in the image. The entrypoint then falls back to `npx tsx` which also fails silently. This fix replaces the inline `tsc` call with a proper package script that fails loudly, and removes the `tsx` fallback path.
Output: Dockerfile compiles seed via `packages/db/package.json` `seed:build` script; entrypoint fails with a useful error if `dist/seed.js` is missing.
</objective>

```typescript

```

The seed uses `../generated/client` — a relative import from `src/seed.ts` pointing to `packages/db/generated/client`.
With `outDir: "dist"` and `rootDir` defaulting to `src`, the compiled `dist/seed.js` would have a broken relative path.
Solution: set `rootDir: "."` so the path `../generated/client` from `src/seed.ts` resolves correctly in the compiled output.
</interfaces>
