Enforce the isLocked constraint on POST /time-entries (the only unguarded write path), add the POST /overtime/unlock-month endpoint with atomic transaction (snapshot delete + entry unlock + audit log), and implement the grace period logic in auto-close-month.

Purpose: Closes the mutation gap where a manager could create a new time entry into a closed month. Adds the admin unlock flow that enables the "unlock → correct → re-close" correction pattern. Prevents premature auto-close during the 14-day correction window.

```typescript
const unlockMonthSchema = z.object({
  employeeId: z.string().uuid(),
  year: z.number().int().min(2020).max(2099),
  month: z.number().int().min(1).max(12),
});

// POST /api/v1/overtime/unlock-month  – Monat entsperren (Snapshot löschen, Einträge entsperren)
app.post("/unlock-month", {
  schema: { tags: ["Überstunden"], security: [{ bearerAuth: [] }] },
  preHandler: requireRole("ADMIN", "MANAGER"),
  handler: async (req, reply) => {
    const { employeeId, year, month } = unlockMonthSchema.parse(req.body);

    // Tenant isolation: verify the employee belongs to the caller's tenant
    const employee = await app.prisma.employee.findUnique({
      where: { id: employeeId },
      select: { tenantId: true },
    });
    if (!employee || employee.tenantId !== req.user.tenantId) {
      return reply.code(404).send({ error: "Mitarbeiter nicht gefunden" });
    }

    const tz = await getTenantTimezone(app.prisma, employee.tenantId);
    const { start: monthStart, end: monthEnd } = monthRangeUtc(year, month, tz);

    // Verify the month is actually closed
    const snap = await app.prisma.saldoSnapshot.findUnique({
      where: {
        employeeId_periodType_periodStart: {
          employeeId,
          periodType: "MONTHLY",
          periodStart: monthStart,
        },
      },
    });
    if (!snap) {
      return reply.code(404).send({ error: "Monat ist nicht abgeschlossen" });
    }

    // D-02/D-03: Atomic transaction — hard-delete snapshot + unlock all non-deleted entries
    await app.prisma.$transaction(async (tx) => {
      await tx.saldoSnapshot.delete({ where: { id: snap.id } });
      await tx.timeEntry.updateMany({
        where: {
          employeeId,
          deletedAt: null,
          date: { gte: monthStart, lte: monthEnd },
        },
        data: { isLocked: false, lockedAt: null },
      });
    });

    // Recalculate live overtime balance now that the snapshot is gone
    await updateOvertimeAccount(app, employeeId);

    // D-02: Audit log — entity SaldoSnapshot, action UNLOCK, oldValue = snapshot data
    await app.audit({
      userId: req.user.sub,
      action: "UNLOCK",
      entity: "SaldoSnapshot",
      entityId: snap.id,
      oldValue: snap,
      request: { ip: req.ip, headers: req.headers as Record<string, string> },
    });

    return reply.code(200).send({ message: "Monat entsperrt" });
  },
});
```

**Part B — overtime.ts earlyClose response (D-12):**

In the `POST /close-month` handler in `overtime.ts`, locate the final `return reply.code(201).send(snapshot);` line (line ~886).

Before it, compute and attach the `earlyClose` informational field:

```typescript
// D-12: Informational hint if the request is made before the grace period ends.
// gracePeriodEnds = the 15th of the month FOLLOWING the target month.
const now = new Date();
const followingMonthDay15 = new Date(Date.UTC(year, month, 15)); // month is 1-based, so month = month+1 in 0-based
const isEarlyClose = now < followingMonthDay15;

const responsePayload: Record<string, unknown> = { ...snapshot };
if (isEarlyClose) {
  responsePayload.earlyClose = true;
  responsePayload.gracePeriodEnds = followingMonthDay15.toISOString();
}

return reply.code(201).send(responsePayload);
```

Note on date calculation: `new Date(Date.UTC(year, month, 15))` — in JavaScript, `Date.UTC(year, month, 15)` uses 0-based months. Since `month` is 1-based (1=January, 12=December), passing `month` directly (without -1) gives the first day of the FOLLOWING month = month+1 in 0-based. Day 15 gives the 15th of the following month. This is the correct calculation.

Example: `year=2025, month=1` (January 2025) → `new Date(Date.UTC(2025, 1, 15))` → Feb 15 2025. Correct.

Replace the original `return reply.code(201).send(snapshot);` with the new conditional send above.
