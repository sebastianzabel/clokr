# Milestone v1.4: Manager/MA-Trennung & Reports

**Status:** ✅ SHIPPED 2026-05-11
**Phases:** 17-25
**Total Plans:** 22

## Overview

v1.4 delivered a clean separation between personal employee views and manager team views,
a modernised Reports page with per-employee DATEV/PDF exports, glass-card polish on the
remaining legacy pages, and a new WiFi-Presence auto-stamping integration with FritzBox.

The milestone goal was: *"Saubere Trennung zwischen persönlichem Bereich und Manager-Ansichten,
Reports-Seite überarbeiten mit Einzel-MA-Exporten."* All 24 requirements (NAV-01..05, TEAM-01..04,
RPT-01..04, UI-01..02, UAT-01..04, WIFI-01..05) shipped wired and verified — see
`v1.4-MILESTONE-AUDIT.md` for the integration map.

Two phases were added after the original v1.4 scope:
- **Phase 24** (UAT Fixes): four findings from real-world UAT — DATEV permission gap, missing
  LODAS Abrechnungszeitraum header, month-filter granularity on Team-Abwesenheiten, and
  manager-on-behalf-of leave creation.
- **Phase 25** (WiFi-Presence Stempel): scope-expansion to add FritzBox TR-064 auto-clock-in/out
  with shift-window gate, opt-in consent flow, and purgeable AuditLog retention (90 days).

## Milestone Summary

**Key Decisions:**

- Personal page cleanup first — removes merge conflict risk for team route additions
- DATEV utility extraction (Phase 21) prerequisite before Reports wires export buttons (Phase 22)
- Glass-card polish (Phase 23) independent of all team/routing work — lowest risk, did last
- TEAM-04 (employee name search filter) assigned to Phase 19 — pattern established there, reused in Phase 20
- Phase 17: Remove unlock button from personal Zeiterfassung entirely — manager-only action that moved to team pages
- Phase 17: Personal time-entries uses `ownEmployeeId` directly with no `selectedEmployeeId` indirection
- Phase 17: iCal team export ungated from `isManager` — all employees can download team absence calendar
- Phase 18: Team nav items inserted before Berichte/Admin so team tools are discoverable first
- Phase 18: Team layout is transparent pass-through (no chrome/tabs) — individual pages own their layout
- Phase 19: Fork personal page rather than build from scratch — ensures full feature parity (ArbZG, lock enforcement, MONTHLY_HOURS, break slots)
- Phase 19: `onMount` defers `loadAll()` until employee selected — no wasted API call on page load
- Phase 19: Manager POST uses `source=CORRECTION` (not `MANUAL`) to distinguish manager corrections in audit trail
- Phase 20: Team leave page forks personal leave page — no submit form, team-wide data, approval code restored from pre-Phase-17 history
- Phase 21: `buildDatevLodas()` kept module-scope (not exported) to avoid leaking DATEV format details outside `reports.ts`
- Phase 23: Promoted NFC-Terminals h3/p title to section-label div to match API Keys reference pattern
- Phase 25: PresenceSource uses soft-delete (`deletedAt`); PresenceDevice uses `onDelete: Cascade` to Employee; Employee.wifiMacs coexists with PresenceDevice for webhook fallback
- Phase 25: `wifiPresenceEnabled=false` enforces GDPR opt-in; `wifiOptInAt` never nulled on opt-out (consent withdrawal trace)
- Phase 25: `purgeable` flag intentionally absent from all wifi consent audit entries — consent events permanently retained
- Phase 25: Hard-delete `purgeable=true` AuditLog rows justified by DSGVO Art. 5(1)(e) — WiFi-presence-only events are not payroll-relevant
- Phase 25: Use minimum tenant retention (most restrictive) as global AuditLog purge cutoff since AuditLog has no tenantId
- Phase 25: `normalizeMac` strip-first approach handles all MAC formats in one pass
- Phase 25: `tenantId_mac` uniqueness — MAC dedup is per-tenant (not per-employee)
- Phase 25: Overnight shift detected by numeric HH:MM comparison; `endDay+1` avoids schema changes
- Phase 25: Presence webhook uses direct `prisma.auditLog.create` (not `app.audit`) for `purgeable` field support
- Phase 25: fritzbox-adapter: `ABSENT_THRESHOLD=6` (6 consecutive polls) balances disconnect responsiveness vs spurious events
- Phase 25: fritzbox-adapter: `GAP_THRESHOLD_MS=30min` — short reconnects are silent to avoid spurious clock-in/out pairs

**Issues Resolved (UAT in Phase 24):**

- DATEV company-wide export rejected MANAGER role (only ADMIN allowed) — fixed by relaxing `requireRole` guard
- LODAS export missing `Abrechnungszeitraum=MMYYYY` and `Beleg_Datum` header — DATEV import wizard failed without these fields
- Team-Abwesenheiten list ignored month filter (year-only granularity) — `calYear/calMonth` not piped into list reload
- Managers could not create leave on behalf of employees — added optional `employeeId` to leave schema with role/tenant gate and modal UI
