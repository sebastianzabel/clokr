## Font Migration Contract (AUDIT-02)

This is the only deliverable with visual implications.

### Source Files to Modify

| File | Change |
|------|--------|
| `apps/web/src/app.css` line 1 | Remove `@import url("https://fonts.googleapis.com/...")`. Replace with `@font-face` blocks. |
| `apps/web/src/hooks.server.ts` lines 70–71 | Remove `https://fonts.googleapis.com` from `style-src` and `https://fonts.gstatic.com` from `font-src`. Set both to `'self'` only. |

### Font Files to Download

Download WOFF2 files to `apps/web/static/fonts/`. Retain all declared weight variants as
decided in D-10. Note: multiple weight files are downloaded to preserve the full font family
for design flexibility, but the UI design contract uses only 2 weights (400 and 600) for
rendered text — see Typography section below.

| Family | Weights | Files |
|--------|---------|-------|
| DM Sans | 300, 400, 500, 600, 700 | `dm-sans-300.woff2`, `dm-sans-400.woff2`, `dm-sans-500.woff2`, `dm-sans-600.woff2`, `dm-sans-700.woff2` |
| Jost | 300, 400, 500, 600, 700 | `jost-300.woff2`, `jost-400.woff2`, `jost-500.woff2`, `jost-600.woff2`, `jost-700.woff2` |
| Fraunces | 300, 400, 600 | `fraunces-300.woff2`, `fraunces-400.woff2`, `fraunces-600.woff2` |

Use WOFF2 format only — no WOFF1 fallback needed (decision from CONTEXT.md specifics).

### @font-face Declaration Pattern

Replace the single `@import` line with one `@font-face` block per weight per family:

```css
@font-face {
  font-family: "DM Sans";
  font-style: normal;
  font-weight: 400;
  font-display: swap;
  src: url("/fonts/dm-sans-400.woff2") format("woff2");
}
```

Apply `font-display: swap` to all declarations (matches Google Fonts `display=swap` behavior).

### CSP Changes

After removing Google Fonts:

- `style-src`: Remove `https://fonts.googleapis.com`. Retain `'self'` and `'unsafe-inline'`.
- `font-src`: Remove `https://fonts.gstatic.com`. Retain `'self'` only.

No new external domains are introduced. Net result: Google domains completely removed from CSP.

### Visual Regression Risk

None expected. Font families, weights, and `font-display: swap` behavior are identical before
and after migration. Rendered output is byte-for-byte equivalent assuming WOFF2 files are the
same font data as served by Google Fonts CDN.

## Typography

No new typography is introduced in Phase 2. The existing scale defined in `app.css` is
authoritative. Documented here for executor reference.

The design contract uses exactly 2 weights: 400 (regular) for body and labels, 600 (semibold)

| Role | Size | Weight | Line Height |

| Display (h1) | 28px (1.75rem) | 600 (semibold) | 1.3 |

## Color

No new color values are introduced in Phase 2. The four-theme system (pflaume, nacht, wald,
schiefer) with CSS custom properties remains unchanged. Documented here for executor reference.

Default theme (pflaume):

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#faf9f7` (`--color-bg`) | Page background |
| Secondary (30%) | `#ffffff` (`--color-surface`) | Cards, sidebar, raised surfaces |
| Accent (10%) | `#80377b` (`--color-brand`) | Active nav items, primary buttons, focus rings, brand links |
| Destructive | `#ef4444` (`--color-danger`) | Delete confirmations, irreversible action buttons only |

Accent reserved for: active navigation state, primary CTA buttons, form focus rings, brand
logo mark. Never used for informational content.

**Source:** `app.css` lines 3–98 (pflaume theme variables).
