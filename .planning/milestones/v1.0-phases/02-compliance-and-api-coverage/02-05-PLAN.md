<interfaces>
<!-- Auth endpoints -->
POST /api/v1/auth/login — { email, password } -> { accessToken, refreshToken }
POST /api/v1/auth/refresh — { refreshToken } -> { accessToken, refreshToken }
Auth middleware: requireAuth (verifies JWT), requireRole(...roles) (verifies role)

<!-- Employee DSGVO anonymization (from RESEARCH.md) -->
DELETE /api/v1/employees/:id — anonymizes instead of hard-deleting:
- Employee: firstName -> "Geloescht", lastName/employeeNumber -> "GELOESCHT-XXX", nfcCardId -> null
- User: email -> anonymized, passwordHash -> "ANONYMIZED", isActive -> false
- Notes in TimeEntries, LeaveRequests, Absences set to null
- AuditLog: userId -> null (anonymized)
- TimeEntries, LeaveRequests, Absences PRESERVED (retention compliance)

<!-- NFC punch -->
POST /api/v1/nfc/punch (or similar — executor must confirm URL)
- Uses terminal API key (not JWT)
- Updates lastUsedAt on NFC card
- Validates API key via SHA256 hash comparison

<!-- Crypto -->
From apps/api/src/utils/crypto.ts:
- encryptIfNeeded(value) — encrypts if not already encrypted (idempotent)
- decryptSafe(value) — decrypts, returns plaintext as fallback if decrypt fails
- Encrypted format: "iv:tag:ciphertext" (3 colon-separated base64 segments, length > 50)

Suffixes: "au" used by auth.test.ts, "em"/"emp" by employees.test.ts, "nfc"/"np" by nfc-punch.test.ts
