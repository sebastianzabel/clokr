# Requirements: Clokr — Production Readiness

**Defined:** 2026-03-30
**Core Value:** Reliable, secure, and legally compliant enough to go live with real customers

## v1 Requirements

Requirements for production launch. Each maps to roadmap phases.

### Test Infrastructure

- [x] **TEST-01**: Isolated test database via `TEST_DATABASE_URL` (no shared dev DB)
- [x] **TEST-02**: Playwright `storageState` setup project (auth once, reuse across E2E specs)
- [x] **TEST-03**: Vitest coverage thresholds enforced (baseline measurement first)
- [x] **TEST-04**: ESLint `no-floating-promises` rule enabled and blocking

### API Test Coverage

- [x] **API-01**: Time entry CRUD tests (create, edit, soft-delete, locked-month rejection, duplicate-day 409)
- [x] **API-02**: Leave request lifecycle tests (request, approve, reject, cancel, cancellation-approve, cross-year booking)
- [x] **API-03**: Overtime saldo and Monatsabschluss tests (saldo read, month-close trigger, locked-month immutability)
- [x] **API-04**: Auth flow tests (login, refresh, JWT expiry, role gates ADMIN/MANAGER/EMPLOYEE)
- [x] **API-05**: DSGVO anonymization tests (no hard delete, all fields anonymized, audit event written)
- [x] **API-06**: NFC terminal punch tests (punch endpoint, lastUsedAt update, API key scoping)

### Compliance & Security

- [x] **SEC-01**: ArbZG compliance unit tests (§3 daily max 10h, §3 24-week average, §4 breaks, §5 rest period, cross-midnight shifts)
- [x] **SEC-02**: Tenant isolation tests (cross-tenant reads/writes blocked on all resources)
- [x] **SEC-03**: Audit trail completeness tests (every mutating endpoint writes AuditLog with required fields)
- [x] **SEC-04**: Soft delete enforcement tests (DELETE sets deletedAt, queries filter deletedAt:null, locked-month entries undeletable)
- [x] **SEC-05**: Timezone/date boundary tests (DST transitions, Dec 31→Jan 1, leap year, month-close at midnight CET vs UTC)

### Code Audit

- [x] **AUDIT-01**: Eliminate all silent `.catch(() => {})` patterns (replace with `app.log.error`)
- [x] **AUDIT-02**: Google Fonts lokal hosten (DSGVO Art. 44 — kein externer Request an Google) (GitHub #100)
- [x] **AUDIT-03**: Docker seed script fix (pnpm@10 + Prisma 7 compatibility) (GitHub #119)

### E2E Tests

- [x] **E2E-01**: Login → Dashboard → Clock in/out flow
- [x] **E2E-02**: Time entry create/edit/delete flow
- [x] **E2E-03**: Leave request → approval → calendar display flow
- [x] **E2E-04**: Admin employee management flow
- [x] **E2E-05**: Monatsabschluss lock/unlock flow (manager)

### UI Quality

- [x] **UI-01**: Mobile-responsive across all views (no horizontal scroll on 390px, 44px touch targets)
- [x] **UI-02**: Consistent error feedback (German-language error messages, retry prompts, locked-month messages)
- [x] **UI-03**: Design consistency audit (spacing, colors, loading skeletons, form components)
- [x] **UI-04**: UX flow improvements (navigation, fewer clicks, clearer feedback patterns)
- [x] **UI-05**: Password policy admin UI (TenantConfig fields exist but no UI)
