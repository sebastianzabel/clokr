## Milestone Summary

**Key Decisions:**

- DATEV LODAS ASCII (not native SSH format) — matches DATEV LODAS import wizard, no BeraterNr/MandantenNr required for v1.1
- iconv-lite for CP1252 — only encoding library with reliable CP1252 in Node ESM context
- OvertimeAccount.balanceHours as stored O(1) read (not recalculate-on-GET) — SALDO-01 architectural decision
- resolvePresenceState() as pure utility (not inline in route) — enables unit testing without DB
- PDFKit streaming via `doc.end() + reply.send(doc)` — avoids Buffer.concat memory spike for 50+ MA reports
- Chart.js `Map<employeeId, Chart>` lifecycle — prevents canvas reuse errors on client-side sort re-renders
- `use:registerCanvas` action replaces `bind:this` function-form — required by rolldown/Vite 8 breaking change

**Issues Resolved:**

- pageAdded infinite recursion in PDFKit streaming fixed with re-entrancy guard
- DATEV .csv→.txt regression from worktree merge overwrites fixed in post-phase cleanup
- WR-01..05 post-verification fixes: tenant timezone in PDFs, year/month validation, revokeObjectURL timing, DATEV label, sick day double-counting
- bind:this function-form build failure (rolldown@1.0.0-rc.15) fixed with use:action pattern

**Issues Deferred:**

- BeraterNr/MandantenNr for full SSH LODAS Native header (DATEV-V2-01) — no confirmed customer need yet
- Abteilungsfilter in dashboard views (RPT-V2-02) — requires Abteilung as first-class entity
- Export-Verlauf audit log integration (RPT-V2-03) — nice-to-have for v1.2+
- Deadline column in Urlaubsübersicht — leave-overview endpoint doesn't select carryOverDeadline

**Technical Debt Incurred:**

- Sparkline reads `carryOver` field (running cumulative saldo) not monthly delta — documented behavior, acceptable for v1.1 trend view
- Leave overview pendingDays uses UTC year boundaries (not tenant timezone) — pragmatic choice, minimal impact for typical usage
- resolvePresenceState priority order hardcoded (not configurable per tenant) — sufficient for v1.1 use cases
