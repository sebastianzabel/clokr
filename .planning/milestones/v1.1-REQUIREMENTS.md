# Requirements: Clokr v1.1 — Reporting & DATEV

**Defined:** 2026-04-11
**Core Value:** Vollständiges Reporting für Inhaber und Manager — Übersichten, Exporte, und DATEV-konformer Lohnexport.

## v1 Requirements

### Saldo Performance & Trend

- [x] **SALDO-01**: Overtime GET endpoint reads stored `OvertimeAccount.balanceHours` without triggering recalculation on every read request — Phase 5 ✓
- [x] **SALDO-02**: `updateOvertimeAccount` is called consistently on all write operations so the stored value is authoritative — Phase 5 ✓
- [x] **SALDO-03**: Manager kann Saldo-Trend der letzten 6 Monate als Sparkline pro Mitarbeiter sehen (basierend auf historischen `SaldoSnapshot`-Einträgen) — Phase 7 ✓

### DATEV Export

- [x] **DATEV-01**: DATEV-Export erzeugt LODAS-kompatibles INI-Format mit `[Allgemein]`, `[Satzbeschreibung]`, `[Bewegungsdaten]` Sektionen — Phase 4 ✓
- [x] **DATEV-02**: DATEV-Export verwendet CP1252-Encoding mit CRLF-Zeilenenden — Phase 4 ✓
- [x] **DATEV-03**: Admin kann Lohnartennummern pro Tenant konfigurieren (Normalstunden, Urlaub, Krank, Sonderurlaub) in den Systemeinstellungen — Phase 4 ✓
- [x] **DATEV-04**: Duplizierte Hilfsfunktionen in `reports.ts` werden vor weiteren Änderungen auf Modulebene extrahiert — Phase 4 ✓

### Reporting Dashboards

- [x] **RPT-01**: Manager sieht Überstunden-Saldo aller Mitarbeiter in einer sortierbaren Tabelle mit Sparkline-Trend — Phase 7 ✓
- [x] **RPT-02**: Manager sieht Urlaubsübersicht aller Mitarbeiter (Resturlaub, genommen, geplant) in einer Ansicht — Phase 7 ✓
- [x] **RPT-03**: Manager sieht Anwesenheitsübersicht des heutigen Tages (da / Urlaub / krank) für alle Mitarbeiter — Phase 7 ✓
- [x] **RPT-04**: Anwesenheitsstatus löst `CANCELLATION_REQUESTED` + `isInvalid`-Einträge korrekt via `resolvePresenceState()` auf (getestete Utility-Funktion) — Phase 5 ✓

### PDF Exports

- [x] **PDF-01**: Firmenweiter Monatsbericht als PDF downloadbar (alle MA, Ist/Soll/Saldo in einer Datei) — Phase 6 ✓
- [x] **PDF-02**: Urlaubsliste als PDF downloadbar (alle MA, alle Urlaubszeiträume, nach Jahr filterbar) — Phase 6 ✓
- [x] **PDF-03**: Monatsbericht-PDF nach Rolle filterbar (Mitarbeiter / Manager / alle) — Phase 6 ✓
- [x] **PDF-04**: Bestehender Einzel-MA-PDF enthält Tenant-Name-Branding und verbessertes Layout — Phase 6 ✓
- [x] **PDF-05**: PDF-Generierung für mehrere Mitarbeiter nutzt Streaming-Response (kein `Buffer.concat`) für 50+ MA — Phase 6 ✓

## v2 Requirements

### DATEV Erweitert

- **DATEV-V2-01**: BeraterNr und MandantenNr in TenantConfig für vollständigen SSH LODAS Native-Format-Header
- **DATEV-V2-02**: DATEV-Export mit automatischer Abruf-Schnittstelle (DATEV Unternehmen online API)

### Reporting Erweitert

- **RPT-V2-01**: Saldo-Trend Sparklines mit echten historischen Monatsdaten (nach SaldoSnapshot vollständig stabilisiert)
- **RPT-V2-02**: Abteilungsfilter in allen Dashboard-Views (wenn Abteilungen als Entity eingeführt werden)
- **RPT-V2-03**: Export-Verlauf (wer hat wann welchen Bericht exportiert — Audit-Log-Integration)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Real-time Anwesenheits-Updates (polling) | UX-Komplexität nicht gerechtfertigt für v1.1 — statischer Snapshot reicht |
| DATEV BeraterNr/MandantenNr | Voller SSH LODAS Native-Header — defer bis Kundenbedarf bestätigt |
| Saldo-Trend-Sparklines ohne SaldoSnapshot | SALDO-01/02/03 sind Voraussetzung — keine Workarounds |
| Gantt-Diagramm Urlaubsplanung | Hohe Komplexität, eigenes Feature-Set — separates Milestone |
| BI/Analytics-Dashboard (Pivot, custom reports) | Eigene Datenbankschicht nötig — falsches Scope für v1.1 |
| Lohnabrechnung / Payslip-Generierung | Gehört in Lohnbuchhaltungs-Software, nicht Zeiterfassung |
