# Requirements: Clokr v1.4

**Defined:** 2026-04-25
**Shipped:** 2026-05-11
**Core Value:** Reliable, secure, legally compliant time tracking SaaS ready for live customers

## v1.4 Requirements

### Navigation & Routing

- [x] **NAV-01**: Personal time-entries page shows only own data (employee selector removed)
- [x] **NAV-02**: Personal leave page shows only own requests (team approval tab removed)
- [x] **NAV-03**: Manager can access dedicated team time-entries page at `/team/time-entries`
- [x] **NAV-04**: Manager can access dedicated team leave page at `/team/leave`
- [x] **NAV-05**: Sidebar MANAGER group shows "Team-Zeiten" and "Team-Abwesenheiten" nav items

### Team

- [x] **TEAM-01**: Manager can view time entries of any team member on team time-entries page
- [x] **TEAM-02**: Manager can create/edit time entries for team members (correction workflow)
- [x] **TEAM-03**: Manager can approve/reject leave requests on team leave page
- [x] **TEAM-04**: Manager can filter employees by name search on team pages

### Reports & Exports

- [x] **RPT-01**: Reports page redesigned with modernized Glass-Design (charts, tables, export buttons)
- [x] **RPT-02**: Reports page has month/year period selector on all widgets
- [x] **RPT-03**: Manager can export single employee as DATEV LODAS TXT
- [x] **RPT-04**: Manager can export single employee as PDF (tabellarischer Stundennachweis)

### UI Polish

- [x] **UI-01**: Schichten page has Glass-Card frame
- [x] **UI-02**: NFC-Terminal overview page has Glass-Card frame

### UAT Fixes (added 2026-05-11 after real-world UAT)

- [x] **UAT-01**: MANAGER role allowed on company-wide DATEV LODAS TXT export endpoint
- [x] **UAT-02**: DATEV LODAS export includes `Abrechnungszeitraum=MMYYYY` and `Beleg_Datum` in `[Allgemein]` header
- [x] **UAT-03**: Team-Abwesenheiten list view filters by month (not year-only)
- [x] **UAT-04**: Manager can create leave requests on behalf of an employee from the team leave page

### WiFi Presence Auto-Stamping (added 2026-05-11)

- [x] **WIFI-01**: FritzBox TR-064 poller detects phone connect/disconnect and posts to `/api/v1/presence/events` webhook
- [x] **WIFI-02**: Admin can map MAC addresses to employees (PresenceDevice model + admin UI); employee can self-manage MACs via "Meine Geräte"
- [x] **WIFI-03**: Clock-in/out only triggers when the WiFi event lies within ±15 min of a scheduled shift start/end (`getCurrentShift` utility)
- [x] **WIFI-04**: GDPR opt-in per employee (`wifiPresenceEnabled` + `wifiOptInAt` consent timestamp); all WiFi-presence-only AuditLog entries are `purgeable=true` and hard-deleted after 90 days
- [x] **WIFI-05**: Admin WiFi-Presence page (key management + device list + opt-in overview) and Profile "Meine Geräte" section both ship

## Out of Scope

| Feature | Reason |
|---------|--------|
| Department/Abteilung filter | Requires Abteilung as first-class entity (schema migration) |
| Excel/CSV export | Diverges from DATEV-first positioning |
| Real-time presence WebSocket | Complexity without proportional value |
| Manager self-approval from team page | Self-approval block is a compliance requirement |
| Server-side route protection | hooks.server.ts never decodes JWTs; client-side onMount guard is the correct pattern |
